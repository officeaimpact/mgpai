# Billing Documentation

Документация по биллингу Yandex Cloud для работы с AI API.

## Структура папки (4 файла)

```
billing/
├── quickstart/
│   └── index.md          # Регистрация аккаунта
├── concepts/
│   ├── billing-account.md # Что такое платежный аккаунт
│   └── limits.md          # Лимиты и квоты
└── operations/
    └── check-charges.md   # Проверка расходов
```

---

## Детальное описание файлов

### `quickstart/index.md` — Регистрация аккаунта в Yandex Cloud

**Когда использовать:** При первоначальной настройке доступа к Yandex Cloud.

**Содержание:**
- Создание платежного аккаунта для физических лиц и юридических лиц
- Получение стартового гранта для новых клиентов
- Пробный период для юридических лиц
- Привязка банковской карты или расчетного счета

**Ключевые моменты:**
- Платежный аккаунт необходим для работы с AI API
- Новые клиенты получают стартовый грант
- Для физлиц — оплата картой или СБП
- Для юрлиц — банковский перевод или карта

---

### `concepts/billing-account.md` — Платежный аккаунт

**Когда использовать:** Для понимания структуры биллинга.

**Содержание:**
- Что такое платежный аккаунт и лицевой счет
- Типы аккаунтов: личный (физлица) и бизнес (юрлица)
- Способы оплаты для каждого типа
- Срок действия аккаунта
- Идентификатор платежного аккаунта

**Типы платежных аккаунтов:**

| Тип | Оплата | Документы |
|-----|--------|-----------|
| Личный | Карта, СБП | Чек на email |
| Бизнес | Банковский перевод, карта | Акт, счет-фактура |

**Важно:**
- Платежный аккаунт может быть привязан к нескольким облакам
- При задолженности более 60 дней — блокировка и удаление данных

---

### `concepts/limits.md` — Квоты и лимиты

**Когда использовать:** Для понимания ограничений сервиса.

**Содержание:**
- Лимиты на количество платежных аккаунтов
- Квоты на использование ресурсов
- Как увеличить квоты

**Ключевые моменты:**
- Лимиты зависят от типа аккаунта
- Можно запросить увеличение квот через поддержку

---

### `operations/check-charges.md` — Детализация расходов

**Когда использовать:** Для мониторинга и контроля затрат на AI API.

**Содержание:**
- Просмотр детализации по облакам, каталогам, сервисам
- Детализация по продуктам и меткам
- Работа с графиками и таблицами
- Фильтрация по периоду

**Глубина детализации:**
- По облакам — какое облако сколько потратило
- По каталогам — расходы внутри облака
- По сервисам — AI Studio, Vision, Search API
- По продуктам — конкретные модели и API
- По меткам — кастомные метки ресурсов

**Показатели:**
- `Стоимость` — без учета скидки
- `К оплате` — итоговая сумма со скидкой

## Типичные сценарии использования

### Начало работы с Yandex AI
1. Прочитать `quickstart/index.md` — создать платежный аккаунт
2. Получить стартовый грант
3. Создать API-ключ (см. `iam/`)

### Мониторинг расходов на AI
1. Открыть `operations/check-charges.md`
2. Выбрать детализацию "По сервисам" → AI Studio
3. Смотреть расходы по продуктам (YandexGPT, эмбеддинги, и т.д.)

### Понимание лимитов
1. Прочитать `concepts/limits.md`
2. При необходимости — запросить увеличение квот

---

## Связь с AI-агентами

AI API тарифицируется по токенам. При разработке AI-агента важно:
- Регулярно проверять расходы через `check-charges.md`
- Понимать лимиты для планирования нагрузки
- Использовать метки для отслеживания расходов по проектам
