## AI и Machine Learning

### `llm.md` — Большие языковые модели (LLM)

**Когда использовать:** Для понимания основ работы YandexGPT и других LLM.

**Содержание:**
- История развития LLM (от GPT-1 до ChatGPT)
- Виды LLM: GPT, BERT, T5
- Архитектура Transformer
- Принцип работы: токенизация → векторизация → генерация
- Примеры использования в бизнесе
- Проблемы LLM: галлюцинации, этика, ограниченные рассуждения

**Ключевые понятия:**
- **Токенизация** — разбиение текста на токены
- **Эмбеддинги** — векторное представление токенов
- **Температура** — параметр "творчества" модели
- **Промпт** — запрос к модели

**Связь с Yandex AI:**
- YandexGPT — основная LLM в AI Studio
- Руководство по промптам: `ai-studio/gpt-prompting-guide/`

### `mcp.md` — Model Context Protocol (MCP)

**Когда использовать:** При интеграции AI-агента с внешними системами через MCP.

**Содержание:**
- Что такое MCP (фреймворк от Anthropic)
- Архитектура: Host → Client → Server
- Роли: user, assistant, system, tools
- Примеры использования: CRM, ERP, базы данных, парсинг
- Сравнение MCP vs бессерверных API
- Проблемы: аутентификация, инъекции промптов

**Ключевые понятия:**
- **MCP-сервер** — связующее звено между LLM и внешними API
- **Tools** — инструменты, доступные агенту
- **Context7** — MCP-сервер для доступа к документации

**Связь с Yandex AI:**
- MCP Hub в AI Studio: `ai-studio/concepts/mcp-hub/`
- Создание MCP-серверов: `ai-studio/operations/mcp-servers/`

## API и протоколы

### `grpc.md` — gRPC

**Когда использовать:** При работе с gRPC API Yandex Cloud.

**Содержание:**
- Что такое gRPC (Google Remote Procedure Calling)
- Составляющие: Protobuf, HTTP/2, IDL
- Режимы работы: Unary, Server streaming, Client streaming, Bidirectional
- Сравнение gRPC vs REST

**Ключевые понятия:**
- **Protobuf** — бинарный формат сериализации данных
- **HTTP/2** — протокол с поддержкой стриминга
- **IDL** — язык описания интерфейса

**Сравнение gRPC и REST:**

| Характеристика | REST | gRPC |
|----------------|------|------|
| Формат | JSON/XML | Protobuf (бинарный) |
| Производительность | Средняя | Высокая |
| Стриминг | Нет | Да |

---

### `rest-api.md` — REST API

**Когда использовать:** При работе с REST API Yandex Cloud.

**Содержание:**
- Принципы REST (Representational State Transfer)
- Методы: GET, POST, PUT, PATCH, DELETE
- Структура запроса: endpoint, параметры, заголовки, тело
- Структура ответа: код, заголовки, тело
- Коды ответов: 2xx, 4xx, 5xx

**Ключевые понятия:**
- **Endpoint** — URL для запроса
- **Headers** — заголовки с метаданными
- **Body** — тело запроса в JSON

---

### `jwt.md` — JSON Web Token

**Когда использовать:** Для понимания аутентификации в API.

**Содержание:**
- Структура JWT: header, payload, signature
- Типы: JWS (подписанный), JWE (зашифрованный)
- Access-токен и Refresh-токен
- Безопасность и уязвимости

**Структура JWT:**
```
header.payload.signature
```

**Связь с Yandex AI:**
- IAM-токены: `iam/concepts/authorization/iam-token.md`
- ID-токены для сервисных аккаунтов

---

### `webhook.md` — Вебхуки

**Когда использовать:** При интеграции AI-агента с внешними событиями.

**Содержание:**
- Принцип работы: событие → HTTP POST → обработка
- Отличие от API (push vs pull)
- Безопасность: HTTPS, HMAC, проверка подписи
- Альтернативы: Message Queue, Polling

**Ключевые понятия:**
- **Push** — сервер отправляет данные автоматически
- **Pull** — клиент запрашивает данные
- **HMAC** — проверка подлинности запроса
